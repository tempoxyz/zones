# Tempo Address Format

This document defines a human-readable hex-based address format for the Tempo ecosystem that supports both mainnet and zone addresses with strong error detection and direct compatibility with standard `0x` hex tooling.

- **TIP ID**: TIP-XXXX
- **Authors/Owners**: Dankrad Feist @dankrad
- **Status**: Draft
- **Related Specs/TIPs**: Zone Validium Design, BIP-350 (bech32m)
- **Protocol Version**: TBD

# Overview

## Abstract

This TIP defines an address format for Tempo that is easily distinguishable from other blockchain addresses, supports zone-specific addresses, and includes strong error detection. The format uses `x`-separated hex components with human-readable prefixes `tempo` (mainnet) and `tempoz` (zones), producing addresses like `tempox00x742d...xf4b7acd9` and `tempozx00x01x742d...xecd8da0f`. A Reed-Solomon checksum over GF(256) guarantees detection of up to 4 hex character substitutions.

## Motivation

As Tempo expands with zones (L2 validiums), users need a clear way to distinguish:
1. Tempo mainnet addresses from other chains (Ethereum, Tron, etc.)
2. Zone addresses from Tempo mainnet addresses
3. Addresses on different zones from each other

A well-designed address format prevents costly mistakes like sending funds to the wrong chain or zone. The `tempo`/`tempoz` prefix provides instant recognition, while the hex encoding preserves direct compatibility with `0x` addresses used by low-level tooling, block explorers, and debugging tools — no base32 encoding or decoding required.

The format uses `x` as the separator between components. Since `x` is not a valid hex character (0-9, a-f), it unambiguously delimits all fields.

# Specification

## Address Structure

A Tempo address consists of `x`-separated components with a Reed-Solomon checksum:

**Mainnet** (HRP `tempo`):

```
tempox{VERSION}x{ADDRESS}x{CHECKSUM}
  │       │         │         │
  │       │         │         └─ 8 hex chars (4-byte RS checksum)
  │       │         └─ 40 hex chars (20-byte raw address)
  │       └─ 2 hex chars (1-byte version)
  └─ Human-readable prefix
```

**Zone** (HRP `tempoz`):

```
tempozx{VERSION}x{ZONEID}x{ADDRESS}x{CHECKSUM}
  │        │        │         │         │
  │        │        │         │         └─ 8 hex chars (4-byte RS checksum)
  │        │        │         └─ 40 hex chars (20-byte raw address)
  │        │        └─ 2+ hex chars (zone ID, big-endian, even-length)
  │        └─ 2 hex chars (1-byte version)
  └─ Human-readable prefix
```

All hex data MUST be lowercase. Decoders SHOULD accept uppercase or mixed-case input by converting to lowercase before validation.

## Human-Readable Prefix (HRP)

The HRP identifies the address type:

| HRP | Prefix in address | Meaning | Has Zone ID |
|-----|-------------------|---------|-------------|
| `tempo` | `tempox` | Tempo mainnet | No |
| `tempoz` | `tempozx` | Tempo zone | Yes |

The HRP is included in the RS checksum computation, so changing the HRP invalidates the address.

## Version Byte

The first data field is a version byte encoded as 2 lowercase hex characters.

| Version | Meaning |
|---------|---------|
| `00` | Current format: 20-byte raw address with optional zone ID |
| `01`–`ff` | Reserved for future use |

Decoders MUST reject addresses with an unrecognized version byte.

## Zone ID Encoding

For zone addresses (HRP `tempoz`), the zone ID is its minimal big-endian hex representation, zero-padded to even length. The `x` separators already delimit the field, so no length prefix is needed.

| Zone ID | Hex encoding | Hex Chars |
|---------|-------------|-----------|
| 1 | `01` | 2 |
| 255 | `ff` | 2 |
| 256 | `0100` | 4 |
| 1000 | `03e8` | 4 |
| 100000 | `0186a0` | 6 |

The encoding MUST be canonical: no unnecessary leading zero bytes. Zone ID 256 is encoded as `0100` (2 bytes), never `000100` (3 bytes). The hex string MUST have even length (zero-padded on the left if the minimal representation has an odd number of hex digits).

For mainnet addresses (HRP `tempo`), the zone ID field is omitted entirely.

### Reserved Zone IDs

- **Zone ID 0**: Reserved for potential future use (e.g., treating Tempo mainnet as "zone 0" for unified cross-zone addressing). The `ZoneFactory` assigns zone IDs starting at 1.
- **Zone IDs 1+**: Assigned sequentially by `ZoneFactory.createZone()`

## Raw Address

The raw 20-byte Ethereum-style address (last 20 bytes of `keccak256` of the public key), encoded as 40 lowercase hex characters.

## Checksum

A 4-byte Reed-Solomon checksum encoded as 8 lowercase hex characters, computed over the HRP, version, zone ID (if present), and raw address. See [Checksum Construction](#checksum-construction) for details.

## Address Length Summary

| Type | Zone ID hex | Total Length |
|------|-----------|--------------|
| Mainnet (`tempox`) | — | **58 chars** |
| Zone (ID ≤ 255) | 2 chars | **62 chars** |
| Zone (ID ≤ 65535) | 4 chars | **64 chars** |
| Zone (ID ≤ 16M) | 6 chars | **66 chars** |
| Zone (ID ≤ 4B) | 8 chars | **68 chars** |

## Checksum Construction

The checksum uses a Reed-Solomon code over GF(256), providing guaranteed detection of up to 4 hex character substitutions and guaranteed correction of up to 2.

### Field Arithmetic

GF(256) is constructed using the primitive polynomial:

```
p(x) = x⁸ + x⁴ + x³ + x² + 1  (0x11D)
```

The primitive element α = 2 has order 255, generating all non-zero elements of GF(256).

### Generator Polynomial

The RS generator polynomial with 4 check symbols is:

```
g(x) = (x − α)(x − α²)(x − α³)(x − α⁴)
     = x⁴ + 0x1e·x³ + 0xd8·x² + 0xe7·x + 0x74
```

where α = 2 in GF(256).

### Checksum Computation

1. Construct the checksum input as a byte sequence:
   - ASCII bytes of the HRP (`"tempo"` or `"tempoz"`)
   - Separator byte: `0x00`
   - Version byte: e.g., `0x00`
   - For zone addresses only:
     - Zone ID length byte: `len(zone_id_bytes)` (1 byte)
     - Zone ID bytes: big-endian minimal encoding
   - Raw address bytes: 20 bytes

2. Compute the RS remainder: divide the input polynomial by g(x) over GF(256) using synthetic division. The 4-byte remainder is the checksum.

3. Encode the 4 checksum bytes as 8 lowercase hex characters.

The HRP bytes are included in the checksum input to ensure that mainnet and zone checksums are never interchangeable. The zone ID length byte ensures the checksum unambiguously commits to the zone ID boundary, even if future versions change the address length.

### Error Detection Properties

The checksum operates at the **byte level** (GF(256) symbols). Each hex character substitution corrupts at most one byte. Therefore:

| Hex char errors | Byte errors | Detection | Correction |
|----------------|-------------|-----------|------------|
| 1 | 1 | Guaranteed | Guaranteed |
| 2 (different bytes) | 2 | Guaranteed | Guaranteed |
| 2 (same byte) | 1 | Guaranteed | Guaranteed |
| 3–4 | ≤ 4 | Guaranteed | If ≤ 2 bytes affected |
| 5+ | Varies | Probabilistic (≈ 1 − 2⁻³²) | — |

The code has minimum distance 5 over GF(256), providing:
- **Guaranteed detection** of any error pattern affecting up to 4 bytes
- **Guaranteed correction** of any error pattern affecting up to 2 bytes

### Minimum Checksum Size Rationale

The Singleton bound requires at least d−1 = 4 check symbols for minimum distance 5. Over GF(256), each symbol is one byte (2 hex chars), giving **8 hex characters** as the minimum provable checksum length for distance 5 at our codeword length. The RS code supports codewords up to 255 bytes, providing ample headroom for any future address size.

## Encoding Algorithm

```python
CURRENT_VERSION = 0

def encode_tempo_address(raw_address: bytes, zone_id: int | None = None, version: int = CURRENT_VERSION) -> str:
    if zone_id is None:
        data = b"tempo\x00" + bytes([version]) + raw_address
        check = rs_checksum(data)
        return "tempox" + format(version, "02x") + "x" + raw_address.hex() + "x" + check.hex()
    else:
        n = max(1, (zone_id.bit_length() + 7) // 8)
        zone_bytes = zone_id.to_bytes(n, "big")
        data = b"tempoz\x00" + bytes([version]) + bytes([len(zone_bytes)]) + zone_bytes + raw_address
        check = rs_checksum(data)
        return "tempozx" + format(version, "02x") + "x" + zone_bytes.hex() + "x" + raw_address.hex() + "x" + check.hex()
```

## Decoding Algorithm

```python
HEX_CHARS = set("0123456789abcdef")

def decode_tempo_address(address: str) -> tuple[bytes, int | None, int]:
    """Returns (raw_address, zone_id, version)."""
    parts = address.lower().split("x")

    if len(parts) == 4 and parts[0] == "tempo":
        hrp = "tempo"
        ver_hex, addr_hex, check_hex = parts[1], parts[2], parts[3]
        zone_hex = None
    elif len(parts) == 5 and parts[0] == "tempoz":
        hrp = "tempoz"
        ver_hex, zone_hex, addr_hex, check_hex = parts[1], parts[2], parts[3], parts[4]
    else:
        raise InvalidFormat()

    if len(ver_hex) != 2 or not all(c in HEX_CHARS for c in ver_hex):
        raise InvalidFormat()
    if len(addr_hex) != 40 or not all(c in HEX_CHARS for c in addr_hex):
        raise InvalidLength()
    if len(check_hex) != 8 or not all(c in HEX_CHARS for c in check_hex):
        raise InvalidLength()
    if zone_hex is not None:
        if len(zone_hex) == 0 or len(zone_hex) % 2 != 0 or not all(c in HEX_CHARS for c in zone_hex):
            raise InvalidFormat()

    version = int(ver_hex, 16)
    raw_address = bytes.fromhex(addr_hex)
    checksum = bytes.fromhex(check_hex)

    if zone_hex is not None:
        zone_bytes = bytes.fromhex(zone_hex)
        if len(zone_bytes) > 1 and zone_bytes[0] == 0:
            raise InvalidFormat()  # non-canonical: leading zero byte
        zone_id = int.from_bytes(zone_bytes, "big")
        data = b"tempoz\x00" + bytes([version]) + bytes([len(zone_bytes)]) + zone_bytes + raw_address
    else:
        zone_id = None
        data = b"tempo\x00" + bytes([version]) + raw_address

    expected = rs_checksum(data)
    if checksum != expected:
        raise InvalidChecksum()

    if version != CURRENT_VERSION:
        raise UnsupportedVersion(version)

    return raw_address, zone_id, version
```

## Validation Rules

1. Split the address string (after lowercasing) by `x`
2. If 4 parts and first is `tempo`: mainnet address
3. If 5 parts and first is `tempoz`: zone address
4. Otherwise: reject
5. Verify version is exactly 2 ASCII hex characters (`[0-9a-f]`)
6. Verify address is exactly 40 ASCII hex characters (20 bytes)
7. Verify checksum is exactly 8 ASCII hex characters (4 bytes)
8. For zone addresses: verify zone ID is an even number of ASCII hex characters with no leading zero byte (canonical encoding)
9. Recompute the RS checksum and compare; reject on mismatch
10. Verify version byte is recognized; reject if unsupported

## Conversion to/from `0x` Addresses

The raw address is directly embedded in the format as hex:

```python
def to_0x(tempo_addr: str) -> str:
    parts = tempo_addr.lower().split("x")
    if parts[0] == "tempo":
        return "0x" + parts[2]
    else:  # tempoz
        return "0x" + parts[3]

def from_0x(hex_addr: str, zone_id: int | None = None, version: int = 0) -> str:
    return encode_tempo_address(bytes.fromhex(hex_addr[2:]), zone_id, version)
```

## Examples

### Tempo Mainnet Address

Raw address: `0x742d35Cc6634C0532925a3b844Bc9e7595f2bD28`

1. HRP: `tempo`
2. Data: version `0x00` (1 byte) + raw address (20 bytes)
3. RS checksum over `b"tempo\x00"` + version + address

### Zone Address (Zone ID = 1)

1. HRP: `tempoz`
2. Data: version `0x00` + zone ID length `0x01` + zone ID `0x01` (1 byte) + raw address (20 bytes)
3. RS checksum over `b"tempoz\x00"` + version + zone ID length + zone ID + address

Result: `tempozx00x01x742d35cc6634c0532925a3b844bc9e7595f2bd28xe05a1a1b` (62 chars)

### Zone Address (Zone ID = 1000)

1. HRP: `tempoz`
2. Data: version `0x00` + zone ID length `0x02` + zone ID `0x03e8` (2 bytes big-endian) + raw address
3. RS checksum over `b"tempoz\x00"` + version + zone ID length + zone bytes + address

Result: `tempozx00x03e8x742d35cc6634c0532925a3b844bc9e7595f2bd28xbdad81ad` (64 chars)

### Zone Address (Zone ID = 100000)

1. HRP: `tempoz`
2. Data: version `0x00` + zone ID length `0x03` + zone ID `0x0186a0` (3 bytes big-endian) + raw address

Result: `tempozx00x0186a0x742d35cc6634c0532925a3b844bc9e7595f2bd28x29b37c41` (66 chars)

# Invariants

1. **Prefix visibility**: Every Tempo address MUST begin with `tempo` or `tempoz` followed by `x`
2. **Separator safety**: Since `x` is not a valid hex character, component boundaries are always unambiguous
3. **Checksum coverage**: The RS checksum covers the HRP, version byte, zone ID length and value (if any), and raw address; changing any of these invalidates the checksum
4. **Version validity**: Decoders MUST reject addresses with unrecognized version bytes
5. **Unique decoding**: Any valid address string MUST decode to exactly one (raw_address, zone_id, version) tuple
6. **Round-trip**: `decode(encode(addr, zone, version)) == (addr, zone, version)` for all valid inputs
7. **Error detection**: Up to 4 hex character substitutions in any position are guaranteed to be detected
8. **Zone 0 reserved**: Zone ID 0 MUST NOT be assigned by ZoneFactory
9. **Case insensitivity**: Addresses MUST be valid regardless of case (but SHOULD be produced in lowercase)
10. **Fixed mainnet length**: All version 0 mainnet Tempo addresses MUST be exactly 58 characters

# Reference Implementation

Complete Python implementation for encoding and decoding Tempo hex addresses. The GF(256) arithmetic uses the primitive polynomial 0x11D with primitive element α = 2.

```python
# === GF(256) with primitive polynomial x^8 + x^4 + x^3 + x^2 + 1 (0x11D) ===

EXP = [0] * 512
LOG = [0] * 256

_x = 1
for _i in range(255):
    EXP[_i] = _x
    LOG[_x] = _i
    _x <<= 1
    if _x & 0x100:
        _x ^= 0x11D
for _i in range(255, 512):
    EXP[_i] = EXP[_i - 255]

def gf_mul(a, b):
    if a == 0 or b == 0:
        return 0
    return EXP[(LOG[a] + LOG[b]) % 255]

# === RS Generator: g(x) = (x - a)(x - a^2)(x - a^3)(x - a^4), a = 2 ===

NSYM = 4

def _make_gen():
    g = [1]
    for i in range(NSYM):
        new_g = [0] * (len(g) + 1)
        ai = EXP[i + 1]
        for j in range(len(g)):
            new_g[j + 1] ^= g[j]
            new_g[j] ^= gf_mul(g[j], ai)
        g = new_g
    return g[::-1]

GEN = _make_gen()  # high-to-low: [0x01, 0x1e, 0xd8, 0xe7, 0x74]

def rs_checksum(data):
    buf = list(data) + [0] * NSYM
    for i in range(len(data)):
        coef = buf[i]
        if coef != 0:
            for j in range(1, len(GEN)):
                buf[i + j] ^= gf_mul(GEN[j], coef)
    return bytes(buf[-NSYM:])

# === Tempo Hex Address ===

CURRENT_VERSION = 0
HEX_CHARS = set("0123456789abcdef")

def zone_id_to_bytes(n):
    length = max(1, (n.bit_length() + 7) // 8)
    return n.to_bytes(length, "big")

def encode_tempo_address(raw_address, zone_id=None, version=CURRENT_VERSION):
    if zone_id is None:
        data = b"tempo\x00" + bytes([version]) + raw_address
        check = rs_checksum(data)
        return "tempox" + format(version, "02x") + "x" + raw_address.hex() + "x" + check.hex()
    else:
        zone_bytes = zone_id_to_bytes(zone_id)
        data = b"tempoz\x00" + bytes([version]) + bytes([len(zone_bytes)]) + zone_bytes + raw_address
        check = rs_checksum(data)
        return "tempozx" + format(version, "02x") + "x" + zone_bytes.hex() + "x" + raw_address.hex() + "x" + check.hex()

def decode_tempo_address(address):
    parts = address.lower().split("x")

    if len(parts) == 4 and parts[0] == "tempo":
        ver_hex, addr_hex, check_hex = parts[1], parts[2], parts[3]
        zone_hex = None
    elif len(parts) == 5 and parts[0] == "tempoz":
        ver_hex, zone_hex, addr_hex, check_hex = parts[1], parts[2], parts[3], parts[4]
    else:
        raise ValueError("Invalid format")

    if len(ver_hex) != 2 or not all(c in HEX_CHARS for c in ver_hex):
        raise ValueError("Invalid version field")
    if len(addr_hex) != 40 or not all(c in HEX_CHARS for c in addr_hex):
        raise ValueError("Invalid address length")
    if len(check_hex) != 8 or not all(c in HEX_CHARS for c in check_hex):
        raise ValueError("Invalid checksum length")
    if zone_hex is not None:
        if len(zone_hex) == 0 or len(zone_hex) % 2 != 0 or not all(c in HEX_CHARS for c in zone_hex):
            raise ValueError("Invalid zone ID")

    version = int(ver_hex, 16)
    addr_bytes = bytes.fromhex(addr_hex)
    check_bytes = bytes.fromhex(check_hex)

    if zone_hex is not None:
        zone_bytes = bytes.fromhex(zone_hex)
        if len(zone_bytes) > 1 and zone_bytes[0] == 0:
            raise ValueError("Non-canonical zone ID: leading zero byte")
        zone_id = int.from_bytes(zone_bytes, "big")
        data = b"tempoz\x00" + bytes([version]) + bytes([len(zone_bytes)]) + zone_bytes + addr_bytes
    else:
        zone_id = None
        data = b"tempo\x00" + bytes([version]) + addr_bytes

    if rs_checksum(data) != check_bytes:
        raise ValueError("Invalid checksum")
    if version != CURRENT_VERSION:
        raise ValueError("Unsupported version: " + str(version))

    return addr_bytes, zone_id, version
```

### Invalid: Mainnet Checksum on Zone Address

A mainnet checksum is invalid for a zone address (and vice versa) because the HRP differs:

```
Valid mainnet: tempox00x742d35cc6634c0532925a3b844bc9e7595f2bd28xf4b7acd9
Invalid zone:  tempozx00x01x742d35cc6634c0532925a3b844bc9e7595f2bd28xf4b7acd9
  → REJECTED: invalid checksum (mainnet checksum used with zone HRP)
```

### Substitution Detection: 2 Characters

```
                  ↓                   ↓
Valid:   tempox00x742d35cc6634c0532925a3b844bc9e7595f2bd28xf4b7acd9
Invalid: tempox00x842d35cc6634c0532925b3b844bc9e7595f2bd28xf4b7acd9
  Changes: '7'->'8', 'a'->'b'
  → REJECTED: invalid checksum
```

### Substitution Detection: 4 Characters

```
                  ↓         ↓         ↓         ↓
Valid:   tempox00x742d35cc6634c0532925a3b844bc9e7595f2bd28xf4b7acd9
Invalid: tempox00xa42d35cc6664c0532925d3b844bc9ea595f2bd28xf4b7acd9
  Changes: '7'->'a', '3'->'6', 'a'->'d', '7'->'a'
  → REJECTED: invalid checksum
```
